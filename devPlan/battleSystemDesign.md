# 战斗系统设计文档

---

## 1. 概述

描述游戏战斗系统的核心规则与交互逻辑。

---

## 2. 建筑系统

### 2.1 建筑状态

| 状态 | 条件 | 行为 |
|------|------|------|
| 正常运行 | 生命值 > 0 且未暂停 | 正常产兵/工作 |
| 受损 | 生命值 > 0 且 < 最大生命 | 继续工作，显示受损视觉 |
| 摧毁 | 生命值 <= 0 | 停止工作，释放格子，可被穿过 |
| 暂停 | 手动暂停 | 停止工作，可恢复 |

### 2.2 建筑摧毁规则

**核心规则**：建筑被摧毁后，其占用的地块恢复空闲状态，士兵可穿过。

- 格子占用释放：建筑摧毁时自动释放所有占用的格子，允许其他单位通行
- 碰撞体保留：保留碰撞体用于检测敌方士兵是否站在建筑上

### 2.3 建筑修复规则

**核心规则**：如果敌方士兵站在被摧毁的建筑上，夜晚商店无法修复该建筑。

- 修复条件：建筑受损、非核心建筑、属于玩家、建筑上无敌方士兵
- 修复成本：按缺失生命值计算（缺失生命 × 每点生命修复金币）

## 3. 士兵系统

### 3.1 士兵行为状态

| 状态 | 条件 | 行为 |
|------|------|------|
| 正常活动 | 存活且活跃 | 移动、追击、攻击 |
| 夜晚休眠 | 存活、无夜战词条、处于夜晚 | 停止移动与攻击 |
| 夜战活动 | 存活、有夜战词条、处于夜晚 | 继续正常活动 |
| 死亡 | 生命值 <= 0 | 淡出后移除 |

### 3.2 士兵与建筑交互

**核心规则**：士兵忽略已摧毁的建筑，可直接穿过。

- 士兵进入建筑碰撞区时，检查建筑是否已摧毁
- 仅对未摧毁的建筑锁定攻击目标
- 攻击目标被摧毁后，自动清除目标引用

### 3.3 目标锁定优先级

1. 敌方士兵（优先）
2. 敌方建筑（其次，仅未摧毁）
3. 敌方城堡（默认推进目标）

## 4. 碰撞层设计

| 层 | 值 | 用途 |
|----|----|------|
| Layer 2 | 2 | 士兵单位 |
| Layer 3 | 4 | 建筑 |

**士兵碰撞配置**：检测士兵 + 建筑
**建筑碰撞配置**：不主动检测其他对象

---

## 5. 昼夜阶段影响

### 5.1 白天阶段
- 出兵建筑正常产兵
- 士兵正常移动、追击、攻击

### 5.2 夜晚阶段
- 无夜战词条的建筑停止产兵
- 无夜战词条的士兵停止行动，原地休眠
- 有夜战词条的单位继续正常行动
- 可在商店修复受损建筑（需满足修复条件）

---

## 6. 关键流程

### 6.1 建筑摧毁流程

```
建筑被摧毁
    ↓
释放格子占用
    ↓
碰撞体保留（用于修复检测）
    ↓
士兵检测到建筑 → 检查是否已摧毁
    ↓
已摧毁 → 忽略，继续移动
未摧毁 → 锁定攻击目标
```

### 6.2 建筑修复流程

```
夜晚商店打开
    ↓
玩家选择修复建筑
    ↓
检查：建筑受损 && 非核心建筑 && 属于玩家
    ↓
检查：建筑上无敌方士兵
    ↓
扣除金币
    ↓
执行修复
```
