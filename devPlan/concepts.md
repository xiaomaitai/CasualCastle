# 游戏概念文档

---

## 昼夜阶段 (Day / Night)

游戏在**白天**与**夜晚**两个阶段间循环，无独立的建造阶段或战斗阶段。详见 `GameManager.GamePhase`。

### 白天阶段 (Day)

**一切正常。**

- 出兵建筑按产出间隔正常产兵
- 士兵正常移动、追击、攻击
- 玩家可打出建筑卡（一次性放置）、进行常规战场交互
- 邻接加成、组合等效果正常生效（完整版）

**实现约定（M1）：**

- 白天为默认初始阶段
- 行为与当前极简 MVP 一致：兵营产兵、士兵战斗、城堡扣血

### 夜晚阶段 (Night)

**经济管理 + 全军休眠（夜战单位除外）。**

- 可**打开商店**购买建筑卡牌（M2）
- 可**花钱修复**受损建筑（M2）
- 可**组合升级**建筑（M5）：**入夜自动**将满足配方、辅材与主体邻接的建筑合并为强化建筑（**免费**）；可用「禁止组合」工具排除指定建筑；规则见 `devPlan/design/combineSystem.md`
- 没有「**夜战**」词条的建筑和士兵**全部休眠**，不再工作：
  - 兵营：停止产兵（`SpawnTimer` 暂停）
  - 士兵：停止移动与攻击，原地待机
- 拥有「夜战」词条的建筑与士兵**豁免休眠**，继续正常行动（M4）

**实现约定（M1）：**

- `GameManager.AdvancePhase()` 在 `Day` ↔ `Night` 间切换，发出 `PhaseChanged` 信号
- `Building.HasNightCombat` / `Soldier.HasNightCombat`：默认 `false`；`true` 时夜晚不休眠
- `NightSystem`（待建）：阶段切换时批量休眠/唤醒单位与建筑

### 夜战 (Night Combat)

**夜战** 是建筑或士兵的词条属性（`HasNightCombat`）。

- 普通单位：夜晚休眠
- 夜战单位：夜晚仍产兵、移动、战斗
- 示例（完整版）：狼人、刺客等特殊单位

## 地块 (Plot)

**地块** 是**城堡内部**放置建筑的最小单元。地块网格**仅限城堡区域**，战场和战斗系统不依赖网格。

- 每个地块有固定的网格坐标 (GridX, GridY)
- 一个建筑可以占据一个或多个**连续的地块**
- 地块的尺寸标准为 1×1 格，每格对应 **100 游戏单位**（渲染时由适配层转换为像素）
- 地块可以处于「空闲」或「已占用」状态
- 地块的朝向固定，不可旋转
- **建筑占满所在格子**；相邻地块之间存在固定间隙，构成士兵通行网络

### 通道系统

城堡内部的地块网格形成天然的通道网络：

- **格间空隙**：相邻地块之间存在固定宽度的间隙（约 15 游戏单位），士兵可在间隙中行走
- **单格建筑**：建筑碰撞体贴合格子边缘；士兵无法穿越建筑所在格子，但可借四周的格间空隙绕过
- **多格建筑**：占用多个连续格子的建筑，内部格子连为一体——**建筑内部无通道**，士兵必须从建筑外围绕行
- **空闲地块**：未放置建筑的格子完全空闲，士兵可自由穿行

**实现约定：**

- `Castle.PlaceBuilding(building, gridX, gridY)` 占用格子并标记为障碍物
- 导航网格随建筑放置/摧毁**动态更新**，已占格子 + 多格建筑内部均为不可行走区域
- `Castle.IsCellPassable(gridX, gridY)` 判断空闲格是否可通行
- 坐标换算：`GetCellCenter(gridX, gridY)` → 本地像素中心

> **网格边界**：地块网格仅用于城堡内部建筑放置和邻接判定。战场上的士兵移动、索敌、碰撞均为**无网格连续空间**，不受地块网格约束。

## 城堡 (Castle / Fort)

**城堡** 是指一个拥有一定数量**地块**的地块集合。

- 标准城堡大小为 **8×8** (64 个地块)
- 战斗中玩家的核心目标：摧毁敌方城堡
- 在一些关卡中城堡可有其他尺寸（如 6×6、10×10 等）
- 城堡由多个地块构成，每个地块上可放置建筑
- 城堡本身有**总生命值**；极简 MVP 中城堡作为整体扣血（`Castle.MaxHealth`）

**实现约定：**

- 预制体：`prefabs/Castle.tscn`，脚本 `Castle.cs`（`Node2D`）
- 城堡**无整体碰撞体**；仅子建筑提供碰撞
- 兵营格子由 `BarracksGridX` / `BarracksGridY` 配置（玩家默认 7,4，敌方 0,4）
- 城堡上方有 `HealthBar`；顶部 UI 血条由 `GameManager` 同步

---

## 建筑 (Building)

**建筑** 是放置在地块上的实体。

- 一个建筑占用 1 个或多个连续地块
- 建筑碰撞体贴合格子边缘（与地块同尺寸，100 游戏单位/格），占满所在格子
- 多格建筑内部所有格子连通，不可穿越；士兵必须从建筑外围绕行
- 建筑拥有属性：生命值、产出间隔等
- 建筑被摧毁后，其占用的地块恢复空闲状态

**实现约定：**

- 基类：`Building.cs`（`Area2D`），碰撞体尺寸 = 占地格数 × 100 游戏单位（适配层转换为像素）
- 碰撞层：**Layer 3（值 4）**；士兵 `collision_mask` 含 4 以检测建筑
- 子类示例：`Barracks` 继承 `Building`，由城堡 `PlaceBuilding` 放入指定格子
- 士兵攻击城堡：进入敌方**建筑**碰撞区后，对父级 `Castle` 调用 `TakeDamage`

---

## 兵营 (Barracks)

出兵建筑，极简 MVP 中双方各有一座，占城堡内单格。

- 每隔固定间隔在**建筑占地框左下角**附近产兵（`GameCoordinates.GetBuildingFootprintSpawnPoint` → `UnitSpawn`）
- 多次产兵在左下角附近轻微错开，避免重叠
- 阵营由所属城堡的 `IsPlayerCastle` 决定
- **夜晚**：无夜战词条时停止产兵；有夜战词条时继续产兵

## 士兵 (Soldier)

极简 MVP 中的基础战斗单位。


| 属性   | 默认值  |
| ---- | ---- |
| 血量   | 30   |
| 伤害   | 10   |
| 速度   | 80   |
| 攻击范围 | 30px |
| 攻击冷却 | 1s   |


**行为：**

- 玩家单位向右、敌方向左推进
- 遇敌追击并攻击；优先攻击敌方士兵，其次攻击敌方城堡（经建筑碰撞触发）
- 可穿过城堡内**空闲格子**；仅在建筑碰撞处停下攻城堡
- 死亡：禁用碰撞、淡出，0.5s 后移除
- **夜晚**：无夜战词条时停止移动与攻击，原地休眠；有夜战词条时行为不变

**碰撞：** Layer 2（值 2），`collision_mask = 6`（检测士兵 + 建筑）

## 组合 (Combine)

**组合** 是入夜时自动将满足配方、且与**主体**四向邻接的材料建筑，合并为一座强化建筑的经济操作（类似背包乱斗同类合成）。

- **时机**：切换到夜晚时**自动执行**；玩家不手动触发组合
- **主体**：配方表 `MainTypeId` 指定；结果落在主体锚点
- **邻接**：每一座辅材须与主体四向相邻；多辅材时关键看是否**都与主体接触**
- **条件**：匹配 `CombineRecipe`、0 阶、非核心、未标记禁止组合；M5 首版仅**单格**建筑
- **代价**：无（不消耗金币）
- **结果**：辅材移除；结果建筑**满血**；属性按配方提升

玩家可通过「禁止组合」工具标记建筑，使其不参与入夜自动组合。完整配方表与验收项见 `devPlan/design/combineSystem.md`。

## 战报 (Battle Report)

**战报** 是玩家一局对战中，每个**夜晚结束**时玩家城堡状态的**快照集合**，是 M6 的核心数据产物。

### 录制

- **时机**：夜晚结束（切换回白天之前），对**玩家城堡**拍摄快照
- **内容**：当时所有已放置建筑的类型、格位、生命值及相关状态
- **序号**：`NightIndex` 从 1 起，第 N 夜结束对应第 N 条快照
- **局内**：先写入本局缓存；**不**自动永久保存

### 永久保存

- **时机**：对局结束、结算弹窗
- 询问玩家**是否记录战报**；确认后**永久保留**（本地持久化）
- 未确认则丢弃本局缓存

### 与 AI（敌方）的关系

敌方**不自主行动**（不购卡、不决策放置）。每局**选定一条**已保存战报作为参考：

- 第 N 夜**开始**时，读取该战报中 `NightIndex == N` 的快照
- 将快照中的建筑**镜像复刻**到敌方城堡对应格位
- 若目标格已被**玩家士兵**占据，则**跳过**该建筑，不复制

完整规则见 `devPlan/design/battleReport.md`（录制与存储）、`devPlan/design/replayAi.md`（镜像复刻）。

## 建筑卡 (Building Card)

**建筑卡** 是商店购买、手牌持有的**一次性放置卡**（当前 M2 已实现）。

- 从商店购入后进入手牌；打出并放置到合法地块后**从手牌移除**
- 手牌中的卡**不与**场上已放置建筑建立绑定或同步状态
- 场上建筑受损、组合、摧毁**不影响**手牌（手牌仅在新购卡时变化）

### 建筑卡 UI 元素（手牌）

- 建筑名称
- 购买费用（商店展示）
- 打出时的放置预览（战场区）

> 已取消「建筑卡持续代表场上建筑、显示实时生命与加成」的完整版设计，不再作为开发目标。

## 战场 (Battlefield)

**战场** 是包含所有地块、建筑和单位的二维**连续空间**（无网格）。

- 战场分为**玩家侧**和**敌方侧**
- 每侧包含一个城堡区域（城堡内部为网格系统）
- 两侧之间为交战区域
- 战场尺寸由关卡配置决定（设计分辨率：1920×1080，主场景 `main_game.tscn`）
- 士兵移动、索敌、碰撞均在连续空间中进行，不受城堡地块网格限制

**视觉呈现（v0.1 草地战场）：**

- **底纹**：绿色草地纹理平铺覆盖整个战场
- **装饰**：随机散布的装饰性草丛精灵（静态、无碰撞），增加视觉层次
- **背景**：山与云的远景图片，位于战场后方形成纵深感（`ParallaxBackground`）
- 玩家城堡、敌方城堡、交战区域的功能位置不变

## 坐标系统

项目有两套坐标，严格分层：

| 坐标系统 | 所在层 | 单位 | 定义 |
|----------|--------|------|------|
| **游戏坐标** | domain 层 | 游戏单位（1 格 = 100 单位） | `GameCoordinateRules.UnitsPerCell = 100` |
| **像素坐标** | adapter 层 | Godot 像素 | `GameCoordinatesAdapter.PixelsPerCell = 72` |

**转换**（仅在 adapter 层）：

```
像素 = 游戏单位 × PixelsPerCell / UnitsPerCell = 游戏单位 × 72 / 100
游戏单位 = 像素 × UnitsPerCell / PixelsPerCell = 像素 × 100 / 72
```

**规则**：
- domain 层所有数值（速度、射程、尺寸、碰撞半径等）一律使用**游戏单位**，禁止出现像素
- adapter 层从 domain 获取游戏单位后，通过 `GameCoordinatesAdapter.GameUnitsToPixels()` 转换为像素再用于渲染 / 物理

## 渲染视角 (2.5D)

项目采用 **2.5D 斜视角纸片人** 的渲染方案：

- **战场** 保持平面网格（2D 俯视坐标），寻路、碰撞、占地判定均在平面上进行
- **角色** 以纸片精灵（billboard sprite）渲染：碰撞体（圆形）位于脚下地面层，显示体（Sprite2D）向上偏移（-Y），形成立体纵深感
- **建筑** 同样为纸片渲染，碰撞体（矩形）贴合地面占地
- **摄像机** 当前为固定视角；未来支持缩放和滚动（见 `devPlan/outline/c08RenderPerspective.md`）

**实现要点：**

| 层级 | 职责 |
|------|------|
| domain | 碰撞半径、显示尺寸均以游戏单位定义，不关心视角 |
| adapter | `ApplyPendingStats()` 中精灵 Y 偏移 = `-displaySize × 0.5`；碰撞体半径 / 尺寸通过 `GameUnitsToPixels()` 转换 |

精灵偏移使角色"站立"在地面碰撞圆上，视觉上脚部与碰撞体重合、身体向上延伸。这与纯 2D 俯视（精灵中心与碰撞中心重合）的渲染方式不同。

## 游戏状态（极简 MVP）

- `GameManager`：管理 `Playing` / `GameOver`、双方血量、胜负
- `UIManager`：顶部血条、结算面板、「返回标题」
- 重开：结算后 `GoToTitle()` 返回标题场景

**完整版扩展（M1 起）：**

- `GameManager.GamePhase`：`Day` / `Night`，提供 `AdvancePhase()` 与 `PhaseChanged` 信号
- 夜晚休眠规则见上文 §昼夜阶段
