# 战斗系统设计文档

---

## 1. 概述

描述游戏战斗系统的核心规则与交互逻辑。

---

## 2. 建筑系统

### 2.1 建筑状态

| 状态 | 条件 | 行为 |
|------|------|------|
| 正常运行 | 生命值 > 0 且未暂停 | 正常产兵/工作 |
| 受损 | 生命值 > 0 且 < 最大生命 | 继续工作，显示受损视觉 |
| 摧毁 | 生命值 <= 0 | 停止工作，释放格子，可被穿过 |
| 暂停 | 手动暂停 | 停止工作，可恢复 |

### 2.2 建筑摧毁规则

**核心规则**：建筑被摧毁后，其占用的地块恢复空闲状态，士兵可穿过。

- 格子占用释放：建筑摧毁时自动释放所有占用的格子，允许其他单位通行
- 碰撞体保留：保留碰撞体用于检测敌方士兵是否站在建筑上

### 2.3 建筑修复规则

**核心规则**：如果敌方士兵站在被摧毁的建筑上，夜晚商店无法修复该建筑。

- 修复条件：建筑受损、非核心建筑、属于玩家、建筑上无敌方士兵
- 修复成本：按缺失生命值计算（缺失生命 × 每点生命修复金币）

## 3. 士兵系统

### 3.1 士兵行为状态

| 状态 | 条件 | 行为 |
|------|------|------|
| 正常活动 | 存活且活跃 | 移动、追击、攻击 |
| 夜晚休眠 | 存活、无夜战词条、处于夜晚 | 停止移动与攻击 |
| 夜战活动 | 存活、有夜战词条、处于夜晚 | 继续正常活动 |
| 死亡 | 生命值 <= 0 | 淡出后移除 |

### 3.2 士兵与建筑交互

**核心规则**：士兵忽略已摧毁的建筑，可直接穿过。

- 士兵进入建筑碰撞区时，检查建筑是否已摧毁
- 仅对未摧毁的建筑锁定攻击目标
- 攻击目标被摧毁后，自动清除目标引用

### 3.3 距离概念

士兵感知敌人依赖三个嵌套距离（由近到远）：

| 距离 | 含义 | 典型值（近战） | 说明 |
|------|------|--------------|------|
| **射程 `AttackRange`** | 可以造成伤害的距离 | 90 px | 边缘距离 ≤ 此值才能攻击 |
| **视野 `VisionRange`** | 士兵自主"看到"敌人的距离 | 250 px | 敌人在视野内 → 主动迎战（交战） |
| **索敌范围** | `BattleManager` 扫描并通知的距离 | 更大 | 敌人在视野外但被告知 → 反击 |

```
         ┌──────────────────────────────────────────────────┐
         │                索敌范围（BattleManager 通知）      │
         │   ┌────────────────────────────────────────┐     │
         │   │         视野 VisionRange               │     │
         │   │   ┌──────────────────────┐             │     │
         │   │   │   射程 AttackRange    │             │     │
         │   │   │   🗡️ 可造成伤害      │             │     │
         │   │   │                      │             │     │
         │   │   └──────────────────────┘             │     │
         │   │   视野内 = 交战                           │     │
         │   └────────────────────────────────────────┘     │
         │   视野外、索敌内 = 反击                            │
         └──────────────────────────────────────────────────┘
```

### 3.4 士兵行为状态机

士兵的每帧行为由以下有限状态机驱动。**核心原则：找不到攻击目标时，总是回到"行军"——向敌方城堡前线推进，即摧毁敌方核心。**

```
                              ┌──────────────────────────────────────────────┐
                              │              行军 MARCHING                   │
                              │   向敌方城堡前线移动（GetDestination）          │
                              │   【默认状态 · 兜底状态】                      │
                              └────┬──────────┬──────────┬─────────────────┘
                                   │          │          │
                    敌方士兵进入    │          │          │ AreaEntered
                    视野范围        │  敌方士兵在视野外   │ 检测到敌方建筑
                    (≤VisionRange)  │  但在索敌范围内     │ 且无更高优先级目标
                                   │  触发"反击"        │
                                   ▼          ▼          ▼
                        ┌──────────────┐ ┌──────────────┐ ┌──────────────┐
                        │ 交战 FIGHTING │ │ 反击 RETAL.  │ │ 攻城 SIEGING │
                        │ 向敌人移动    │ │ 向敌人移动    │ │ 攻击阻挡建筑  │
                        │ 射程内攻击    │ │ 射程内攻击    │ │ 伤害穿透城堡  │
                        └──────┬───────┘ └──────┬───────┘ └──────┬───────┘
                               │                │                │
                目标死亡/      │  目标进入视野   │  目标死亡/     │
                移出索敌范围    │  → 升级为交战   │  移出索敌范围   │
                               │                │                │
                               ▼                │                │
                              ┌──────────────────────────────────────────────┐
                              │              行军 MARCHING                   │
                              │          （回到核心任务）                      │
                              └──────────────────────────────────────────────┘

        攻城中的士兵发现敌方士兵（视野内）→ 交战
        攻城中的士兵发现敌方士兵（视野外）→ 反击
        反击中的士兵发现敌方士兵进入视野 → 交战（升级）
        反击中的士兵撞到建筑（AreaEntered）→ 攻城（被障碍阻挡）
        建筑被摧毁 且 无其他目标 → 行军
```

#### 状态定义

| 状态 | 含义 | 移动方式 | 攻击行为 |
|------|------|----------|----------|
| **行军 `Marching`** | 无攻击目标，执行核心任务：摧毁敌方城堡 | `NavigationAgent` 驶向 `GetDestination()`，遇建筑滑行绕行 | 无 |
| **交战 `Fighting`** | 敌方士兵在**视野范围内**，主动迎战 | 向目标移动，拉近距离 | 进入射程后 `_attackBehavior.TryExecute()`，冷却就绪时造成伤害 |
| **反击 `Retaliating`** | 敌方士兵在**视野外、索敌范围内**，主动出击 | 向目标移动，拉近距离 | 进入射程后 `_attackBehavior.TryExecute()`，冷却就绪时造成伤害 |
| **攻城 `Sieging`** | 攻击挡路的敌方建筑 | 停在建筑旁 | 每冷却周期对建筑造成 `Damage` 点伤害，穿透至城堡 |

#### 状态转换规则

| 转换 | 触发条件 | 说明 |
|------|----------|------|
| **行军 → 交战** | 敌方士兵进入 `VisionRange` | 视野内发现敌人，主动迎战 |
| **行军 → 反击** | `BattleManager` 索敌周期（0.2s）通知视野外有敌方士兵 | 视野外但知道敌人在哪，主动出击 |
| **行军 → 攻城** | `AreaEntered` 检测到敌方建筑，且无士兵目标 | 建筑阻挡去路，拆障 |
| **交战 → 行军** | 目标死亡，**或** 目标移出索敌范围 | 威胁消失，回到核心任务 |
| **交战 → 反击** | 目标移出 `VisionRange` 但仍 ≤ 索敌范围 | 目标跑出视野但还在追踪范围内，降级为反击继续追 |
| **反击 → 交战** | 目标进入 `VisionRange` | 追上了，升级为交战 |
| **反击 → 行军** | 目标死亡，**或** 目标移出索敌范围 | 追丢了，放弃 |
| **反击 → 攻城** | `AreaEntered` 检测到敌方建筑 | 追击途中被建筑阻挡 |
| **攻城 → 交战** | 敌方士兵进入 `VisionRange` | 士兵优先级 > 建筑 |
| **攻城 → 反击** | 敌方士兵在视野外但索敌范围内 | 士兵优先级 > 建筑 |
| **攻城 → 行军** | 建筑被摧毁，且无其他目标 | 障碍消失，继续向城堡推进 |
| **任意 → 行军** | 当前目标无效（死亡/销毁/`null`），且无其他候选目标 | **兜底规则：总是回到摧毁敌方核心** |

#### 状态机伪代码

```
每帧 _Process(dt):
  1. 检查存活、游戏状态、昼夜
     → 不存活 / 游戏结束 / 休眠 → return
  2. 扣减攻击冷却
  3. 确定当前状态:

     if _targetEnemy != null && _targetEnemy.IsAlive:
         dist = DistanceTo(_targetEnemy) - 双方碰撞半径
         if dist <= VisionRange:
             state = FIGHTING
         else:
             state = RETALIATING   // 视野外但索敌范围内

     else if _targetBuilding != null && !_targetBuilding.IsDestroyed
           && _targetCastle != null && _targetCastle.IsAlive:
         state = SIEGING

     else:
         state = MARCHING

  4. 执行状态行为:
     FIGHTING / RETALIATING:
         if 边缘距离 <= AttackRange:
             _attackBehavior.TryExecute(target)
         else:
             NavigationAgent → target.GlobalPosition  // 拉近距离
     SIEGING:
         building.TakeDamage(Damage), 穿透至 Castle
     MARCHING:
         NavigationAgent → GetDestination()
```

#### 目标优先级与兜底

```
目标优先级（从高到低）:
  1. 视野内的敌方士兵     → FIGHTING（主动迎战）
  2. 视野外的敌方士兵     → RETALIATING（主动出击）
  3. 阻挡路径的敌方建筑    → SIEGING（攻城拆障）
  4. 无目标               → MARCHING（向敌方城堡前线推进）

兜底规则:
  - 任何状态下，若当前目标消失且无新候选，立即转为 MARCHING
  - MARCHING 的目的地永远指向敌方城堡方向（GetDestination）
  - 反击与交战的本质相同（向敌人移动、射程内攻击），区别仅在于触发距离和优先级
  - 交战优先级高于反击：当同时存在视野内和视野外的敌人时，优先迎战视野内的
  - 士兵的最终目的始终是摧毁敌方核心，战斗只是清除障碍
```

## 4. 碰撞层设计

| 层 | 值 | 用途 |
|----|----|------|
| Layer 2 | 2 | 士兵单位 |
| Layer 3 | 4 | 建筑 |

**士兵碰撞配置**：检测士兵 + 建筑
**建筑碰撞配置**：不主动检测其他对象

## 5. 昼夜阶段影响

### 5.1 白天阶段
- 出兵建筑正常产兵
- 士兵正常移动、追击、攻击

### 5.2 夜晚阶段
- 无夜战词条的建筑停止产兵
- 无夜战词条的士兵停止行动，原地休眠
- 有夜战词条的单位继续正常行动
- 可在商店修复受损建筑（需满足修复条件）

## 6. 关键流程

### 6.1 建筑摧毁流程

```
建筑被摧毁
    ↓
释放格子占用
    ↓
碰撞体保留（用于修复检测）
    ↓
士兵检测到建筑 → 检查是否已摧毁
    ↓
已摧毁 → 忽略，继续移动
未摧毁 → 锁定攻击目标
```

### 6.2 建筑修复流程

```
夜晚商店打开
    ↓
玩家选择修复建筑
    ↓
检查：建筑受损 && 非核心建筑 && 属于玩家
    ↓
检查：建筑上无敌方士兵
    ↓
扣除金币
    ↓
执行修复
```
