# 六、界面和 UI 设计

本章描述目标界面结构，不要求在 M1.1 直接实现全部 UI。M1.1 的目标是把主界面、次要界面、位置关系、尺寸比例、联动规则和特殊交互定义清楚，供 M2 商店与手牌实现时对齐。

百分比坐标约定：

- 以游戏视口为基准，左上角为 `(0%, 0%)`，右下角为 `(100%, 100%)`。
- 位置使用 `x / y / w / h` 表示：左上角横坐标、左上角纵坐标、宽度、高度。
- 百分比是布局目标，不是必须逐像素严格实现；1280×720 作为当前参考分辨率。
- UI 覆盖层使用屏幕百分比定位；战场内容使用世界坐标，只有战场相机变化时才跟随缩放或移动。

## 6.2 标题界面

标题界面是游戏启动后的第一屏。底层背景不再使用静态图，而是一场持续进行的演示战斗：我方和敌方各自使用预设的高级城堡，不断出兵并在中间战线交战，摄像头随战线在左右方向缓慢偏移。上层保留正常菜单和 UI 按钮，优先保证游戏名、开始游戏和退出入口清晰可读。标题界面不显示局内资源、阶段、血量或手牌。

标题演示战场规则：

- 演示战场只服务标题界面的氛围展示，不参与真实局内进度、金币、昼夜阶段、手牌或胜负结算。
- 我方和敌方高级城堡使用专门的标题界面预设，允许包含尚未在正式关卡开放的高级建筑外观与出兵组合。
- 双方持续自动出兵，单位在中间战线交战；战斗循环应能长期运行，不出现一方永久推平导致画面停止。
- 摄像头根据当前主要交战位置在左右方向缓慢偏移，偏移范围限制在能同时暗示双方城堡存在的区间内。
- 标题演示战场可以使用独立简化数据，避免复用真实对局状态导致开始游戏时残留单位、计时器或信号连接。

前景菜单规则：

- 标题、开始按钮和退出按钮始终位于演示战场上方，并受背景压暗层保护。
- 鼠标悬浮或点击菜单按钮时，按钮反馈优先于背景战斗表现。
- 开始游戏后停止或释放标题演示战场，再切换到游戏战场界面。
- 从结算界面返回标题后，重新创建或重置标题演示战场。

联动规则：

- 点击开始按钮后切换到游戏战场界面，并初始化一局游戏。
- 点击退出按钮后关闭游戏。
- 从结算界面返回标题后，应能重新开始一局。
- 标题界面播放 BGM，但不绑定局内状态。

金币显示规则：

- 金币区固定在顶部状态栏左中部，靠近玩家血量，避免和手牌区费用显示混淆。
- 金币数变化时可短暂闪烁或跳动，但不遮挡战场。
- 金币不足导致卡牌或商店商品不可用时，金币区可短暂变红提示。

阶段联动规则：

- `GameManager` 阶段变化时，阶段显示和倒计时同步刷新。
- 白天阶段：阶段显示使用暖色或浅色。
- 夜晚阶段：阶段显示使用冷色或深蓝紫色。
- 游戏结束时顶部状态栏保留，但输入焦点转交给结算面板。

特殊规则：

- 战场区是所有局内 UI 的底层，默认不遮挡顶部状态栏和手牌区。
- 未来支持滚动时，拖动战场区空白处可移动相机；UI 覆盖层保持固定。
- 未来支持缩放时，鼠标滚轮缩放战场内容；顶部状态栏、手牌区、商店区不缩放。
- 战场相机缩放范围建议控制在 80% 到 140%，避免 UI 与战场比例失衡。
- 当商店、暂停菜单或结算面板打开时，战场点击输入被屏蔽，但战场画面仍可作为背景显示。

交互联动：

- 鼠标悬浮单位或建筑时，触发单位/建筑信息面板。
- 选中手牌中的建筑卡后，战场区进入放置预览状态。
- 放置预览状态下，己方地块显示合法/非法颜色。
- 游戏结束后，战场区不再响应放置、选中和商店相关输入。

- 金币不足、阶段不允许或无合法地块时，卡牌显示不可用状态。
- 商店打开时，手牌区仍可见且可继续操作；购买与放置可并行进行。

## 6.12 次要界面优先级

界面叠加优先级从高到低：

1. 结算面板
2. 暂停/菜单界面
3. 商店区 / 后续融合界面
4. 放置预览层
5. 单位/建筑信息面板
6. 手牌区、顶部状态栏、金币区、菜单按钮
7. 战场区

高优先级界面打开时，可以临时屏蔽低优先级界面的输入，但不应隐藏必要的局内状态信息。结算面板是唯一完全接管局内输入的界面。

## 6.14 M2 实现状态与 M3 UI 建议

M2 已完成最小可用实现：

1. 顶部金币区显示金币数。
2. 手牌区支持 3 到 5 张卡，点击选中或拖拽放置。
3. 商店右侧面板展示 3 个商品槽，支持购买入牌与拖拽直放。
4. 放置预览层显示己方地块合法/非法颜色。
5. 菜单界面与信息面板延后到 M3。

M3 建议优先补强：

1. 多建筑类型的卡牌图标与商店展示。
2. 建筑悬浮/选中信息面板。
3. 邻接加成在地块或建筑上的视觉反馈。
