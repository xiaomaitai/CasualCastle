# RTS 单位卡牌 UI 设计（V1）

## 设计目标

采用 2D 萌系、粗线条、高辨识度的美术风格，以卡牌代替传统 RTS 单位模型在地图上活动。

设计原则：

- 信息层级清晰，优先保证战场快速识别
- 卡图占据主要视觉面积，提高单位辨识度
- 尽量减少分隔线，仅保留必要结构
- 所有信息均可在缩放状态下快速读取

## 卡牌结构

卡牌整体为正方形。

```
┌────────────────────────┐
│        生命值条         │
│        单位名称         │
├────────────────────────┤
│                    状态 │
│                        │
│                        │
│       单位插画         │
│                        │
│                        │
│                        │
│                        │
│ Buff Buff Buff Buff    │
└────────────────────────┘
```

![单位卡牌示例](unitCardExample.png)

> 卡牌整体结构示意：顶部为生命值条与单位名称，下方大面积为插画区，状态图标位于右上角，Buff 图标位于左下角。

整体由两个区域组成：

1. **顶部信息区**
2. **单位插画区**

两部分之间保留一条横向分隔线，其余区域不再使用额外分隔线。

## 外边框

采用双层边框设计。

| 层级 | 样式 | 说明 |
|------|------|------|
| 外层 | 深色粗描边、圆角 | 保证地图背景复杂时仍具有清晰轮廓 |
| 内层 | 浅色细描边 | 与插画区域形成缓冲，提高整体质感 |

## 顶部信息区

顶部信息区包含生命值与单位名称。

### 生命值条

位于最顶部，占满整个宽度。

特点：

- 横向铺满
- 高度适中
- 根据当前生命值实时缩放长度
- 不显示生命值数字

生命值条颜色：

| 血量状态 | 颜色 |
|---------|------|
| 满血 | 绿色 |
| 中等 | 黄色 |
| 残血 | 红色 |

### 单位名称

位于生命值条内居中显示。

- 白色字体、黑色粗描边
- 居中对齐
- 字体保持统一高度

由于文字直接覆盖在生命条上，因此无论生命值变化都应保证足够的可读性。

## 单位插画区

插画区占据卡牌绝大部分面积，为视觉主体。约占卡牌总面积 **80%**。

插画要求：

- Q版萌系
- 粗黑描边
- 高饱和配色
- 正面或 45° 角展示
- 动作具有攻击性或职业特征
- 留有一定边距，不贴边框

插画区域增加一圈细边框作为画框。背景保持简单：浅色渐变 + 少量装饰图案，不使用复杂纹理。

### 状态图标

位于插画区右上角。

- 同时仅显示一个状态
- 无状态时隐藏
- 不占用额外布局空间
- 圆形底、高饱和颜色
- 尺寸略大于 Buff 图标
- 与插画保持一定间距

状态示例：受击、激怒、睡眠、冰冻、眩晕、沉默、中毒

### Buff 图标区

位于插画区左下角。

- 水平排列
- 最多显示 4 个
- 超出数量时最后显示 `+N`

示例：

```
🛡 ⚔ ❄ ☠
```

或

```
🛡 ⚔ ❄ +3
```

Buff 图标采用统一规格：圆形底、彩色图标、深色描边，与状态图标保持统一视觉风格。

## 分隔线

仅保留一条横向分隔线：

```
生命区
────────────
插画区
```

Buff 区与状态区均直接覆盖在插画区域内，不再单独划分区域。

## 信息优先级

```
单位插画
    ↓
生命值
    ↓
单位名称
    ↓
状态
    ↓
Buff
```

保证玩家第一眼识别单位类型，其次关注生命状态，再关注特殊状态与增益效果。

## 推荐比例

| 区域 | 建议占比 |
|------|---------|
| 生命值条 | 8% |
| 名称区 | 10% |
| 插画区 | 82% |

插画区内部：

- 状态图标：右上角
- Buff 图标：左下角

## 视觉规范

![阵营卡牌示例](unitCardFactionExample.png)

> 不同阵营（种族）的卡牌应在保持统一结构的前提下，通过配色、边框装饰、插画风格区分阵营特征。

整体风格：

- 萌系
- 粗线条
- 高对比
- 高辨识度
- 中高饱和配色

边框：

- 外边框较粗
- 内边框较细

字体：

- 白字
- 黑色粗描边

阴影：

- 轻微投影
- 不使用复杂发光效果

整体视觉应适合 RTS 场景下大量单位同时存在，保证在缩放视角下依然能够快速识别单位类型、生命状态与异常状态。
