# 融合系统设计 — M5

夜晚将满足配方、且与**主体建筑**四向邻接的材料建筑，自动合并为一座强化建筑（类似背包乱斗的同类合成升级）。概念入口见 `concepts.md`；架构归属见 `outline/c05ProjectArchitecture.md`。

---

## 时机与触发

- **仅夜晚**：在 `GameManager` 切换到 `Night` 时，由 `FusionSystem` **自动扫描并执行**所有合法融合
- **局内条件**：`Playing` 且未暂停、未结算
- **仅玩家侧**：敌方城堡建筑不参与融合
- **无融合工具按钮**：玩家不手动点选「融合」；入夜后系统自动完成
- **禁止融合工具**：底部工具栏提供「禁止融合」按钮（与暂停、修复并列），用于标记不参与自动融合的建筑

---

## 基本规则

### 配方与主体

每条 `FusionRecipe` 指定：

| 字段 | 说明 |
|------|------|
| `MainTypeId` | **主体**建筑类型（融合结果落在此建筑锚点） |
| `MaterialTypeId` | 辅材建筑类型 |
| `MaterialCount` | 除主体外、须与主体四向邻接的辅材数量 |
| `GoldCost` | 本次融合消耗金币 |
| `ResultTypeId` | 结果建筑类型 |

**邻接判定（多建筑时）：**

- 主体 1 座，辅材 `MaterialCount` 座
- 每一座辅材须与**主体**四向相邻（与 `AdjacentSystem` 一致）
- 辅材之间**不要求**互相邻接
- 材料须匹配配方类型；M5 首版 `MainTypeId == MaterialTypeId`（同类双融）

### 材料条件

参与融合的建筑须**同时满足**：

| 条件 | 说明 |
|------|------|
| 归属 | 玩家城堡内已放置建筑 |
| 状态 | 未摧毁、非手动暂停 |
| 类型 | 非核心建筑（不可融合城堡之心） |
| 阶数 | M5 首版仅 **0 阶 → 1 阶**，已 1 阶不可再融 |
| 禁止标记 | 未标记「禁止融合」 |
| 占据 | 未被敌军站在建筑格上（与修复规则一致） |
| 配方 | 能与主体组成某条 `FusionRecipe` 的完整材料组 |
| 金币 | 玩家金币 ≥ 配方 `GoldCost` |

### 占地限制（M5 首版）

- **仅支持单格建筑**（兵营、狼穴）
- 多格建筑（靶场、马厩）**暂不纳入** M5

### 融合结果

- 结果建筑落在**主体**的锚点格
- 所有辅材移除，其占格恢复空闲
- 结果建筑 **满血**（`Health = MaxHealth`），不继承材料生命比例
- 产兵进度：**清零**
- 手动暂停、禁止融合标记：**不继承**（结果建筑为默认可操作状态）
- 融合后调用 `AdjacentSystem.RefreshCastle` 刷新邻接加成

### 经济

- 每次融合消耗金币，数额由配方定义
- 金币不足时，该组材料**跳过**（本夜不融合），不扣费

### 入夜自动执行顺序

```text
PhaseChanged → Night
  → FusionSystem.ResolveNightFusions(castle)
       循环直到无可融合组：
         扫描城堡内所有合法 (主体 + 辅材) 组
         按稳定规则选取一组（见下）→ TryFuse → 刷新邻接
  → 继续夜晚流程（开商店等）
```

**选取规则（实现约定，避免歧义）：**

- 优先处理「主体锚点」字典序最小的一组（`AnchorGridY`，再 `AnchorGridX`）
- 同一建筑同一夜最多参与一次融合
- 一次 `TryFuse` 只消耗一条配方的一组材料

## 禁止融合

### 工具交互

- 底部「禁止融合」按钮：toggle 工具模式（与暂停 / 修复互斥，`ButtonGroup`）
- 激活后左键点击玩家侧、非核心建筑：切换该建筑的 `IsFusionProhibited`
- 右键 / Esc：退出工具
- **白天也可用**（提前标记，入夜生效）；与修复工具不同，不限制夜晚

### 表现

- 被禁止的建筑在格子上显示可辨识标记（如禁止图标，实现时与暂停图标并列或叠加）
- 建筑信息面板悬停时可显示「已禁止融合」

### 语义

- `IsFusionProhibited == true` 的建筑**不能**作为主体或辅材
- 取消禁止后，下一夜入夜扫描可正常参与融合

## M5 配方表（首版）

| 主体 | 辅材（须邻接主体） | 数量 | 费用 | 结果 | 相对 0 阶提升 |
|------|-------------------|------|------|------|----------------|
| 兵营 | 兵营 | 1 | 8 | 强化兵营 | 生命 130；产兵间隔 4s（原 5s） |
| 狼穴 | 狼穴 | 1 | 10 | 强化狼穴 | 生命 120；产兵间隔 5s；狼人攻击 16 |

**命名约定：** 结果建筑在 `BuildingSystem` 中作为独立 `TypeId`（`BarracksT2`、`WolfDenT2`）。

## 系统职责

| 模块 | 职责 |
|------|------|
| `FusionSystem` | 配方表、`FindFusibleGroups` / `TryFuse`、`ResolveNightFusions`、扣费、移除辅材与生成结果 |
| `Building` | `IsFusionProhibited` 字段及状态图标 |
| `BuildingSystem` | 强化建筑定义、`CreateBuilding`、占地与产兵属性 |
| `FusionProhibitUiController` | 禁止融合工具按钮、点击切换标记、光标 |
| `GameManager` | 入夜时调用 `FusionSystem.ResolveNightFusions` |
| `ShopSystem` | 仅提供 `TrySpendGold`；融合费用不走商店 |
| `UIManager` | 注册禁止融合工具的 `Process` / `HandleInput` |

信号建议：`FusionCompleted(Castle, Building result)`，供 UI 音效或日志。

## 边界与不做事项

| 场景 | 处理 |
|------|------|
| 白天 | 不执行融合；可标记禁止融合 |
| 材料被敌军占据 | 该建筑不参与融合 |
| 游戏中途暂停 | 入夜融合在暂停前已执行；禁止融合工具退出 |
| 手牌卡牌 | 手牌为**一次性放置卡**，打出后消耗，**不与场上建筑绑定**；融合不改变手牌 |
| 融合动画 | 可选：简短缩放/闪光；无动画亦可通过验收 |
| AI 融合 | M6；M5 仅玩家 |

---

## 验收标准（M5）

- [ ] 入夜时，两座邻接同类型单格兵营（未禁止、够金币）自动融合为强化兵营
- [ ] 结果落在主体锚点，辅材移除，结果**满血**
- [ ] 扣费正确；金币不足时本组不融合
- [ ] 「禁止融合」工具可标记 / 取消标记；被标记建筑不参与自动融合
- [ ] 白天不融合；核心建筑、多格建筑、已 1 阶建筑不可误融
- [ ] 融合后邻接加成正确刷新
- [ ] 与商店 / 修复 / 手牌 / 昼夜循环输入不冲突
