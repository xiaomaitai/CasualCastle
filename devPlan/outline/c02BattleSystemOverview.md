# 二、战斗系统：版本概述

---

## 2.1 现有状况

### 单位

单一 `Soldier` 类（`Area2D`），所有属性通过 `[Export]` 硬编码：

| 属性 | 默认值 | 说明 |
|------|--------|------|
| Health | 30 | 归零死亡 |
| Damage | 10 | 每次攻击伤害 |
| Speed | 80 | 像素/秒 |
| AttackRange | 30 | 像素，近战范围 |
| AttackCooldown | 1.0s | 攻击间隔 |

建筑数据（`BuildingDefinitions`）可通过 `SoldierDamage` / `SoldierSpeed` / `SoldierHealth` / `SoldierAttackRange` 覆写产出士兵的属性。但所有士兵行为完全相同——只有数值差异，没有行为差异。

### 移动方式

直线推进，无寻路：

- 默认方向：玩家单位向右（`Vector2.Right`），敌方向左（`Vector2.Left`）
- 目标点：`GlobalPosition + _moveDirection * 1000`（屏幕外远点）
- 遇敌时 `MoveToward(dt, enemy.GlobalPosition)` 直接冲向敌人
- 无避障、无网格约束、无队形

### 碰撞检测

基于 Godot `Area2D` 信号（`AreaEntered` / `AreaExited`）：

- Layer 2：士兵（值 2）
- Layer 3：建筑（值 4）
- 士兵 `collision_mask = 6`（检测士兵 + 建筑）
- 每个士兵独立接收信号，无全局管理器

**问题：** 士兵数量增多时 O(n²) 信号风暴；同一帧内多个士兵同时进入/离开区域时信号顺序不确定。

### 目标选择

信号驱动，无主动索敌：

```
AreaEntered
  → 如果是敌方士兵 → _targetEnemy = other（优先）
  → 如果是敌方建筑 → _targetCastle = castle, _targetBuilding = building

AreaExited
  → 离开的是当前目标 → 清空目标
```

**问题：**
- 目标切换无优先级比较——先进先锁
- 无法主动选择最优目标（最近的、血最少的等）
- 敌人死亡/建筑摧毁后需要依赖 AreaExited 信号才能切目标，有延迟
- 无「目的地」概念——不知道敌方城堡在哪，只能盲推

### 战斗循环（_Process）

```
1. 检查存活、游戏状态、昼夜
2. 扣减攻击冷却
3. 有敌方士兵目标：
   - 在射程内 → 攻击
   - 在射程外 → 追击
4. 无士兵目标，有建筑目标：
   - 攻击建筑 → 伤害经建筑传到城堡
5. 无任何目标：
   - 向默认方向盲推
```

### 死亡

- `IsAlive = false` → 禁用碰撞 → 半透明 → 0.5s 后 `QueueFree`
- 无死亡动画、无音效、无尸体

## 2.2 实现目标

### 寻路系统

**当前：** 直线移动，遇障碍不绕行。士兵走到建筑背面就卡住。

**目标：**

1. **网格导航**：战场划分为通行/阻塞网格（基于城堡空地 + 已摧毁建筑），士兵沿网格最短路径移动
2. **目的地设定**：每个士兵知道自己的目的地——玩家单位目的地为「敌方城堡正前方战线」，敌方单位目的地为「玩家城堡正前方战线」
3. **动态重算**：建筑被摧毁后，其占格变为可通行，附近士兵重新规划路径
4. **性能**：使用流场（flow field）或 A* 分帧计算，避免每帧全量寻路

**技术选择：**
- 流场适合大量单位同向移动（双方士兵分别流向对方城堡），O(grid) 而非 O(units × grid)
- 流场每 0.5s 刷新一次，或建筑摧毁时触发刷新
- 单位移动时采样所在格的流向，沿流向移动

### 碰撞优化

**当前：** `AreaEntered/AreaExited` 信号，每个士兵独立接收。N 个士兵 = N × M 次信号触发。

**目标：**

1. **全局管理器替代信号**：`BattleManager` 每帧或每 N 帧扫描一次所有单位，统一更新目标
2. **空间分区**：将战场划分为网格桶，每个桶记录其中的士兵和建筑。索敌时只扫描邻近桶
3. **碰撞响应分离**：物理碰撞（防止堆叠）与索敌检测（目标选择）分两条路径
4. **单位上限控制**：限制场上最大单位数（如 200），超出时不再产兵

**具体方案：**

- 建立 `BattleManager`（Node），持有所属方所有士兵列表
- 每 0.2s 更新一次索敌：对每个士兵，查询其所在网格桶及相邻桶中的敌方单位，取最近者
- 士兵间物理推挤：使用 `move_and_slide` 或简单排斥力，防止堆叠在同一位置
- 移除 `AreaEntered/AreaExited` 信号依赖

### 战斗 AI

**当前：** 单位无决策逻辑——遇敌就打，没敌就盲推。

**目标：**

#### 目的地

每个士兵维护一个 `Destination`（世界坐标），按优先级决定：

| 优先级 | 目的地 | 条件 |
|--------|--------|------|
| 1 | 当前攻击目标位置 | 已有目标且在追击中 |
| 2 | 敌方城堡正前方战线位置 | 无目标的默认目的地 |
| 3 | 行军方向远点 | 流场尚未计算时的 fallback |

**战线位置计算：**
- 玩家方战线：`playerCastle 右边缘 + 100px`
- 敌方战线：`enemyCastle 左边缘 - 100px`
- 双方士兵在战线相遇形成交战带

#### 目标选择

替代 AreaEntered 信号。`BattleManager` 定期对每个士兵执行：

```
1. 在索敌范围内（AttackRange × 态度系数）扫描敌方单位
2. 按优先级排序：
   a. 可攻击的敌方士兵（在 AttackRange 内 > 在追击范围内）
   b. 可攻击的敌方建筑（在 AttackRange 内）
   c. 最近的敌方单位（追击目标）
3. 选择最优目标
```

**态度系数：**
- 近战单位：`scanRange = AttackRange * 2`（短距扫描，优先接敌）
- 近战单位：`scanRange = AttackRange * 3`（中距扫描）

#### 目标切换

单位不盲锁首个进入范围的目标。在以下时机重新评估：

1. 当前目标死亡/摧毁 → 立即切换到下一个最优目标
2. `BattleManager` 索敌周期（0.2s）→ 比较当前目标和新候选，切换条件：
   - 新目标优先级更高
   - 当前目标距离超过追击放弃距离（如 300px）
3. 建筑被摧毁 → 攻击该建筑的士兵立即重选目标

#### 攻击决策

```
攻击前检查：
  - 目标存活？
  - 目标在 AttackRange 内？
  - 攻击冷却就绪？
  - 未处于昼夜休眠？

攻击执行：
  - 近战：立即造成伤害（CombatRules.ApplyDamage）
  - 远程：创建弹体飞向目标（到达时造成伤害）
```

## 2.3 验收标准

- [ ] 单位沿流场网格到达敌方战线，不再卡在建筑背后
- [ ] 建筑摧毁后通行，周围单位自动绕行
- [ ] 200 单位同屏时帧率 ≥ 60fps（碰撞优化验证）
- [ ] 单位优先攻击最近的威胁，而非盲锁首个进入范围的敌人
- [ ] 当前目标死亡后 0.2s 内切换到下一个目标
- [ ] 近战和远程单位行为有实质差异
- [ ] `dotnet build` 0 错误，现有 6 个领域测试通过
