# 四、功能清单

## MVP（已完成）

| 模块 | 功能 | 状态 | 备注 |
|------|------|------|------|
| 游戏流程 | 白天/夜晚交替、阶段切换 | ✅ | 60s 白天 / 30s 夜晚，支持手动跳过 |
| 夜晚经济 | 商店购卡、花钱修复建筑 | ✅ | 含拖拽直放与建筑修复 |
| 夜晚休眠 | 无夜战词条的单位/建筑停止工作 | ✅ | 夜战单位豁免 |
| 夜战单位 | 狼穴产狼人，夜晚行动 | ✅ | M4 |
| 商店系统 | 购买建筑卡牌 | ✅ | 5 选项，含拖拽直放 |
| 手牌系统 | 卡牌管理、打出 | ✅ | 7 张手牌，点击与拖拽放置 |
| 建筑系统 | 多类型、多格占地、产出、修复 | ✅ | 兵营/靶场/马厩/狼穴 |
| 战斗系统 | 多兵种、远程、克制、血条、死亡动画 | ✅ | 含伤害类型/护甲/矩阵框架 |
| 邻接系统 | 建筑邻接加成 | ✅ | 兵营邻接加速 + 光圈 |
| 融合系统 | 入夜自动融合、禁止融合工具 | ✅ | 仅单格建筑；见 `devPlan/design/fusionSystem.md` |
| UI 界面 | 商店、手牌、金币、信息面板、暂停、设置 | ✅ | 含开发者模式开关 |
| AI 对手 | 战报镜像复刻（无自主决策） | ✅ | M6，见 `devPlan/design/replayAi.md` |
| 战报系统 | 夜末快照、结算永久保存 | ✅ | M6，见 `devPlan/design/battleReport.md` |
| 架构 | 六边形分层 + 4 domain 项目 + 双层 DI | ✅ | 含 AdapterRegistry + MS DI |
| 数据层 | 建筑/单位 SQLite 配置 | ✅ | `config.db`，`SqliteBuildingRepository` + `SqliteUnitRepository` |
