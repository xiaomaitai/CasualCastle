### P1 草地战场

当前战场为灰色单色背景。v0.1 替换为绿色草地场景：

- **草地底纹**：绿色草地 tile 或平铺纹理，覆盖整个战场区域
- **装饰草**：在草地上随机散布装饰性草丛精灵（静态，无碰撞），增加视觉层次
- **背景**：战场后方放置山和云的远景图片（ParallaxBackground 或静态 Sprite），形成纵深感
- 战场功能区域（玩家城堡、敌方城堡、中间交战区）位置不变

实现方式：
- 草地 tile：`TextureRect` 或 `TileMap` 平铺
- 装饰草：`MultiMeshInstance2D` 随机散布（性能友好）
- 山云背景：`ParallaxBackground` + `ParallaxLayer` + `Sprite2D`

### P2 建筑与通道系统

核心规则变更——建筑与格子的关系：

- **建筑占满格子**：一个建筑覆盖其占地范围内的所有格子，碰撞体贴合格子边缘
- **格间空隙**：相邻格子之间存在固定的通行间隙（约 15 游戏单位），士兵可在空隙中行走
- **多格建筑不可穿越**：多格建筑内部所有格子连为一体，中间无通道，士兵必须绕行
- **通行规则**：士兵寻路时，已放置建筑的所有格子 + 建筑内部均为障碍物；格间空隙构成自然路径网络

对现有系统的影响：
- 建筑碰撞体从当前的"略小于格子"改为贴合格子边缘（100 游戏单位/格）
- 寻路导航网格（NavigationRegion2D / TileMap navigation layer）需随建筑放置/摧毁动态更新
- `OccupancyGrid` 逻辑不变——已占格子就是障碍物

格间空隙示意（俯视）：
```
┌──────┐ ┌──────┐
│ 建筑A │ │ 建筑B │
│ 2×2  │ │ 2×2  │
│      │ │      │
└──────┘ └──────┘
  ↑ 空隙 ↑ 空隙 ↑  ← 士兵可通行
```

### P4 组合系统扩展

当前 M5 组合仅支持单格建筑（`IsCombinableMaterial` 检查 `Footprint.Length == 1`）。P4 扩展：

- 移除 `Footprint.Length == 1` 限制，允许多格建筑参与组合
- 多格建筑组合规则：主体与辅材的占地大小一致即可；结果建筑沿用主体占地
- 邻接判定适配多格：辅材的**任意格子**与主体的**任意格子**四向相邻即满足邻接条件
- 人类 3 条配方：兵营+兵营→高级兵营、靶场+靶场→高级靶场、马厩+马厩→高级马厩
- 组合后结果满血、产兵进度清零、邻接刷新

### P5 邻接加成扩展

当前仅兵营有邻接加速。P5 扩展到所有人类建筑：

| 建筑类型 | 邻接条件 | 效果 |
|---------|---------|------|
| 兵营 | 与兵营/高级兵营四向邻接 | 产兵间隔 -15% |
| 靶场 | 与靶场/高级靶场四向邻接 | 产兵间隔 -15% |
| 马厩 | 与马厩/高级马厩四向邻接 | 产兵间隔 -15% |

- 同类型建筑链（Tier 1 与 Tier 2 互认同类型）
- 加成可叠加（两座互邻建筑各自获得 -15%）
- 光环可视化沿用现有 `AdjacentLinkPulse`

### P6 数据层完善

当前 `config.db` 已存储建筑和单位定义。P6 扩展：

- `building_defs` 表新增人类 6 条建筑记录（含 footprint JSON、组合阶数）
- `unit_stats` 表新增人类 6 条单位记录（含伤害类型、护甲类型、体型）
- `combine_recipes` 表新增 3 条配方
- `damage_matrix` 表（新建）：4×4 伤害/护甲倍率矩阵
- `DamageMatrix.LoadFrom()` 从 SQLite 读取倍率数据，替换当前全 1.0 默认值
- 商店卡池表（新建）：每种建筑的商店出现权重与费用

技术约定：数据存储统一使用 SQLite（`assets/data/config.db`），非必要不使用 `.tres` 资源文件或 `.tscn` 预制体存储配置数据。

### P7 存档与打磨

**存档系统**：
- 保存内容：建筑布局（类型+格位+血量）、金币、昼夜轮次、当前手牌、关联战报 ID
- 保存时机：手动保存（暂停菜单）+ 自动保存（每夜结束时）
- 存储格式：SQLite 数据库（`user://saves/`，一个存档一个 `.db` 文件）
- 读取：标题界面「继续游戏」按钮，读取最新存档恢复游戏状态

**摄像机系统**：
- 滚轮缩放（限制最小/最大缩放比）
- 中键或右键拖拽平移
- 边界限制（摄像机不超出战场范围）
- Home 键重置视角

**平衡打磨**：
- 基于测试数据调优产兵间隔、单位属性、金币经济
- 伤害矩阵在实战中验证并调整倍率
