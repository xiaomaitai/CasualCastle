### P1 草地战场

当前战场为灰色单色背景。v0.1 替换为绿色草地场景：

- **草地底纹**：绿色草地 tile 或平铺纹理，覆盖整个战场区域
- **装饰草**：在草地上随机散布装饰性草丛精灵（静态，无碰撞），增加视觉层次
- **背景**：战场后方放置山和云的远景图片（ParallaxBackground 或静态 Sprite），形成纵深感
- 战场功能区域（玩家城堡、敌方城堡、中间交战区）位置不变

实现方式：
- 草地 tile：`TextureRect` 或 `TileMap` 平铺
- 装饰草：`MultiMeshInstance2D` 随机散布（性能友好）
- 山云背景：`ParallaxBackground` + `ParallaxLayer` + `Sprite2D`

### P2 建筑与通道系统

核心规则变更——建筑与格子的关系：

- **建筑占满格子**：一个建筑覆盖其占地范围内的所有格子，碰撞体贴合格子边缘
- **格间空隙**：相邻格子之间存在固定的通行间隙（约 15 游戏单位），士兵可在空隙中行走
- **多格建筑不可穿越**：多格建筑内部所有格子连为一体，中间无通道，士兵必须绕行
- **通行规则**：士兵寻路时，已放置建筑的所有格子 + 建筑内部均为障碍物；格间空隙构成自然路径网络

对现有系统的影响：
- 建筑碰撞体从当前的"略小于格子"改为贴合格子边缘（100 游戏单位/格）
- 寻路导航网格（NavigationRegion2D / TileMap navigation layer）需随建筑放置/摧毁动态更新
- `OccupancyGrid` 逻辑不变——已占格子就是障碍物

格间空隙示意（俯视）：
```
┌──────┐ ┌──────┐
│ 建筑A │ │ 建筑B │
│ 2×2  │ │ 2×2  │
│      │ │      │
└──────┘ └──────┘
  ↑ 空隙 ↑ 空隙 ↑  ← 士兵可通行
```

### P4 组合系统扩展

当前 M5 组合仅支持单格建筑（`IsCombinableMaterial` 检查 `Footprint.Length == 1`）。P4 扩展：

- 移除 `Footprint.Length == 1` 限制，允许多格建筑参与组合
- 多格建筑组合规则：主体与辅材的占地大小一致即可；结果建筑沿用主体占地
- 邻接判定适配多格：辅材的**任意格子**与主体的**任意格子**四向相邻即满足邻接条件
- 人类 3 条配方：兵营+兵营→高级兵营、靶场+靶场→高级靶场、马厩+马厩→高级马厩
- 组合后结果满血、产兵进度清零、邻接刷新

### P5 邻接加成扩展

当前仅兵营有邻接加速。P5 扩展到所有人类建筑：

| 建筑类型 | 邻接条件 | 效果 |
|---------|---------|------|
| 兵营 | 与兵营/高级兵营四向邻接 | 产兵间隔 -15% |
| 靶场 | 与靶场/高级靶场四向邻接 | 产兵间隔 -15% |
| 马厩 | 与马厩/高级马厩四向邻接 | 产兵间隔 -15% |

- 同类型建筑链（Tier 1 与 Tier 2 互认同类型）
- 加成可叠加（两座互邻建筑各自获得 -15%）
- 光环可视化沿用现有 `AdjacentLinkPulse`

### P6 数据层完善

当前 `config.db` 已存储建筑和单位定义。P6 扩展：

- `building_defs` 表新增人类 6 条建筑记录（含 footprint JSON、组合阶数）
- `unit_stats` 表新增人类 6 条单位记录（含伤害类型、护甲类型、体型）
- `combine_recipes` 表新增 3 条配方
- `damage_matrix` 表（新建）：4×4 伤害/护甲倍率矩阵
- `DamageMatrix.LoadFrom()` 从 SQLite 读取倍率数据，替换当前全 1.0 默认值
- 商店卡池表（新建）：每种建筑的商店出现权重与费用

技术约定：数据存储统一使用 SQLite（`assets/data/config.db`），非必要不使用 `.tres` 资源文件或 `.tscn` 预制体存储配置数据。

### P7 存档与打磨

**存档系统**：
- 保存内容：建筑布局（类型+格位+血量）、金币、昼夜轮次、当前手牌、关联战报 ID
- 保存时机：手动保存（暂停菜单）+ 自动保存（每夜结束时）
- 存储格式：SQLite 数据库（`user://saves/`，一个存档一个 `.db` 文件）
- 读取：标题界面「继续游戏」按钮，读取最新存档恢复游戏状态

**摄像机系统**：
- 滚轮缩放（限制最小/最大缩放比）
- 中键或右键拖拽平移
- 边界限制（摄像机不超出战场范围）
- Home 键重置视角

**平衡打磨**：
- 基于测试数据调优产兵间隔、单位属性、金币经济
- 伤害矩阵在实战中验证并调整倍率

### A2 AI 自主决策

当前敌方行为完全依赖战报回放——仅镜像复刻玩家历史操作。v0.2 实现自主决策 AI。

**决策层次：**

| 层次 | 决策内容 | 触发时机 |
|------|---------|---------|
| 经济 | 购买哪张建筑卡 | 夜晚商店阶段 |
| 放置 | 建筑放在哪个格子 | 购买后 / 手牌有空时 |
| 组合 | 是否禁止某建筑参与组合 | 夜晚组合阶段 |
| 修复 | 是否修复受损建筑 | 夜晚经济阶段 |

**AI 难度分级：**

| 难度 | 说明 |
|------|------|
| 简单 | 随机购买 + 随机合法放置，适合新手 |
| 普通 | 偏好同类型抱团 + 优先修复 + 简单组合策略 |
| 困难 | 针对性兵种搭配 + 最优放置 + 主动凑组合 |

**实现要点：**
- `IReplayTarget` 接口扩展或新增 `IAiDecisionPort` 端口
- AI 决策在 domain 层完成（纯 C#，不依赖 Godot 节点）
- 放置决策：扫描己方城堡空格，按启发式评分选最优格位
- 购买决策：基于当前金币、手牌空间、已有建筑类型分布计算权重
- 组合决策：评估组合收益 vs 失去建筑的损失
- 可插拔难度配置（`ai_difficulty` 表或 config）

**验收标准：**
- [ ] 敌方在夜晚自动购买卡牌并放置建筑
- [ ] 敌方建筑产出士兵并推进战斗
- [ ] 敌方能完成 T1+T1→T2 组合（不要求 T3）
- [ ] 三种难度行为有明显差异
- [ ] AI 不破坏游戏流程（不崩溃、不放非法位置）

### A3 战斗表现深化

当前战斗缺乏视觉反馈——弹体不可见、无受击/死亡特效、皇家兵种特殊能力没有视觉区分。

**弹体可视化：**
- 远程单位（弓箭手/弩手/神射手）发射可见弹体
- 弹体：小型 Sprite2D，从攻击方飞向目标，命中后消失
- 弹体速度 = 攻击方到目标距离 / 0.3s（快速但不瞬间）
- 弹体命中时触发目标受击闪白（已有 `SoldierVisual` 受击闪白能力）

**死亡特效：**
- 单位 HP 归零时播放消散动画（0.3s 淡出 + 缩小）
- 建筑摧毁时播放碎片/烟尘粒子

**皇家兵种能力视觉：**
- 皇家禁卫横扫：攻击时短暂弧形刀光
- 皇家盾卫回复：绿色治疗数字弹出
- 皇家骑士冲锋：移动速度激增时身后拖尾
- 皇家斥候潜行：半透明渲染

**实现要点：**
- 弹体：`Projectile` 预制体（Area2D + Sprite2D），由 `SoldierLogic` 或 `BattleManager` 管理
- 死亡动画：`SoldierLifecycle` 扩展淡出逻辑
- 能力特效：程序化粒子/刀光，不依赖外部素材

**验收标准：**
- [ ] 远程单位发射可见弹体，命中后消失
- [ ] 单位死亡有淡出消散动画
- [ ] 皇家兵种能力有视觉区分

### A4 数值平衡

T3 皇家兵种引入后需要全兵种横向平衡，以及 4→1 建筑消耗的经济合理性。

**经济模型：**
- 一座 T3 = 4 座 T1 的投入（4 次购买 + 4 次放置 + 3 次组合等待）
- T3 单体战力应约等于 3~4 座 T1 的持续产兵总和
- T3 占地效率更高（1 座 2×2 vs 4 座 2×2 = 节省 12 格）

**平衡参数：**

| 维度 | 范围 | 说明 |
|------|------|------|
| T3 产兵间隔 | 8~16s | 皇家斥候最快，皇家骑士最慢 |
| T3 建筑 HP | 200~500 | 皇家壁垒最高，皇家游骑最低 |
| 皇家兵种单体战力 | T2 的 1.5~2.5x | 按兵种定位浮动 |
| 金币经济 | 不变 | T3 不通过商店，组合免费 |

**测试重点：**
- 纯 T1 铺场 vs 冲 T2 vs 冲 T3 三种策略胜率
- 五条线的选择分布（是否有明显最优解）
- AI 各难度对玩家的胜率

**验收标准：**
- [ ] 三种策略（T1 铺场/T2 中盘/T3 后期）各有所长，无明显通解
- [ ] 五条线选择率相对均衡
- [ ] 玩家与普通 AI 胜率约 50%
