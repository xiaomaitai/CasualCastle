# 六、界面和 UI 设计

本章先描述目标界面结构，不要求在 M1.1 直接实现全部 UI。M1.1 的目标是把主界面、次要界面、位置关系和联动规则定义清楚，供 M2 商店与手牌实现时对齐。

## 6.1 主界面总览

| 主界面 | 使用场景 | 组成界面 | 当前状态 |
|------|----------|----------|----------|
| 标题界面 | 启动游戏、进入对局、退出游戏 | 标题区、开始按钮、退出按钮、背景音乐 | 已实现 |
| 游戏战场界面 | 一局对战的主要操作界面 | 持续战场区、顶部状态栏、手牌区、商店区、单位/建筑信息面板、结算面板 | 部分实现 |
| 结算界面 | 胜负产生后展示结果和返回入口 | 结果文本、返回标题按钮 | 已实现 |

## 6.2 标题界面

标题界面是游戏启动后的第一屏，整体保持简单居中布局。标题文字位于画面上方或中部，开始 / 退出按钮位于标题下方。标题界面播放 BGM，点击开始后切换到游戏战场界面。

特殊规则：

- 标题界面不承载局内状态。
- 从结算界面返回标题后，应能重新开始一局。

## 6.3 游戏战场界面

游戏战场界面是一局游戏的主界面，由持续战场区和多个覆盖式 UI 区域组成。战场本身始终存在，商店、信息面板、结算面板等次要界面按状态叠加显示。

### 持续战场区

持续战场区位于画面中央，占据主要屏幕空间，用于展示双方城堡、建筑、士兵、弹道或特效。当前项目仍是固定视角；完整版目标可支持横向滚动、缩放或镜头拖动，但这些能力需要等战场尺寸扩大后再实现。

联动规则：

- 白天和夜晚都显示战场区。
- 白天单位正常移动和战斗，夜晚普通单位休眠。
- 点击或悬浮战场中的单位/建筑时，可打开单位/建筑信息面板。
- 选择手牌中的建筑卡后，战场区进入放置预览状态。

特殊规则：

- 战场区是所有局内 UI 的底层。
- 如果未来支持滚动和缩放，UI 覆盖层不跟随镜头缩放，战场内容跟随镜头变化。

### 顶部状态栏

顶部状态栏固定在屏幕上方，显示双方城堡血量、当前阶段、阶段倒计时，以及开发用跳过阶段按钮。

联动规则：

- 订阅 GameManager 的血量、阶段和游戏状态变化。
- 阶段切换时更新阶段文字和倒计时。
- 游戏结束后停止强调倒计时，将焦点转交给结算面板。

### 手牌区

手牌区位于屏幕底部，横向排列玩家当前持有的建筑卡。每张卡显示名称、费用和可打出状态。M2 会实现基础手牌区，M1.1 只定义位置和交互规则。

联动规则：

- 从商店购买卡牌后，卡牌加入手牌区。
- 点击卡牌后，战场区进入建筑放置预览。
- 放置成功后，对应卡牌从手牌区移除。
- 金币不足、阶段不允许或没有可放置地块时，卡牌显示不可用状态。

### 商店区

商店区是可由 UI 按钮打开的覆盖式面板，优先在夜晚使用。默认建议从右侧滑出或居中弹出，显示可购买卡牌、刷新入口、金币消耗和关闭按钮。

联动规则：

- 夜晚允许打开和购买，白天默认隐藏或禁用。
- 购买成功后扣除金币，并把卡牌加入手牌区。
- 打开商店时战场仍可显示，但可以暂停或弱化非关键输入，避免误操作。

### 单位/建筑信息面板

单位/建筑信息面板在鼠标悬浮或选中战场对象时显示，优先放在鼠标附近或屏幕侧边，展示名称、生命值、攻击力、是否夜战、工作状态等简要信息。

联动规则：

- 悬浮对象变化时刷新内容。
- 鼠标离开对象或打开更高优先级面板时隐藏。
- 夜晚休眠时显示休眠状态，夜战单位显示可行动状态。

### 放置预览层

放置预览层在玩家选中建筑卡后显示，覆盖在己方城堡地块上。合法地块高亮为可放置，非法地块显示禁用样式。

联动规则：

- 来源是手牌区的卡牌选中状态。
- 鼠标移动时更新预览位置。
- 点击合法地块后调用建筑放置逻辑；成功后关闭预览并移除手牌。
- 取消选中、打开商店或游戏结束时关闭预览。

### 结算面板

结算面板是游戏结束后的高优先级覆盖界面，居中显示胜负结果和返回标题按钮。

联动规则：

- 由 GameManager 的游戏结束状态触发。
- 显示时屏蔽商店、手牌打出和战场放置输入。
- 返回标题后清理本局 UI 状态。

## 6.4 次要界面优先级

界面叠加优先级从高到低：

1. 结算面板
2. 商店区 / 后续融合界面
3. 放置预览层
4. 单位/建筑信息面板
5. 手牌区、顶部状态栏
6. 持续战场区

高优先级界面打开时，可以临时屏蔽低优先级界面的输入，但不应隐藏必要的局内状态信息。
