# 七、修订后的开发顺序

> 以下顺序基于 M0 已完成的前提重新编排。已跳过的任务（项目初始化、基础场景、极简战斗）不再重复列出。

## 7.1 第一阶段：游戏流程框架（M1，约 1 周）— ✅ 已完成并验证

**目标：** 从「一直产兵对战」升级为「白天 / 夜晚」两阶段循环骨架。

| 任务 | 描述 | 预估 |
|------|------|------|
| 阶段状态机 | 在 `GameManager` 增加 `Day` / `Night` 及 `AdvancePhase()` | 1.5 天 |
| 阶段 UI | 顶部阶段指示器、开发用「切换昼夜」按钮 | 0.5 天 |
| 夜晚休眠门控 | 夜晚暂停无夜战词条的兵营产兵与士兵行动 | 1 天 |
| 夜战词条占位 | `Building` / `Soldier` 增加 `HasNightCombat` 属性（默认 false） | 0.5 天 |
| 数据资源骨架 | 创建 `GameConfig`、首批 `BuildingData` .tres | 1 天 |
| 目录搭建 | 建立按业务模块划分的 `scripts/` 子目录与 `resources/` 子目录 | 0.5 天 |

**交付物：**

- 可手动切换 Day ↔ Night 的 Demo
- 白天行为与当前 M0 一致
- 夜晚双方兵营与士兵休眠（尚无夜战单位）
- 数据结构文件就位，供 M2 使用

**目标：** 在进入 M2 实现前，先定义游戏整体主界面、战场内次要界面、布局位置、联动关系和特殊交互规则。

| 任务 | 描述 | 预估 |
|------|------|------|
| 主界面枚举 | 列出标题界面、游戏战场界面、结算界面 | 0.5 天 |
| 战场 UI 拆分 | 列出持续战场区、顶部状态栏、手牌区、商店区、信息面板等次要界面 | 0.5 天 |
| 界面规则说明 | 描述位置、样式、联动、优先级和战场滚动/缩放等特殊规则 | 0.5 天 |
| 线框图和场景图 | 补充各界面线框图、主要场景示意图和场景流转图 | 0.5 天 |

**交付物：**

- `outline/c06UiDesign.md`
- M2 商店与手牌实现前的 UI 设计依据
- 标题场景、战场常态、商店、建筑放置、暂停菜单、结算场景和场景流转示意图
