# 八、下一步开发内容（立即行动）

**当前焦点：M1.1 界面和 UI 设计。** 在进入 M2 实现前，先把游戏主界面、次要界面、位置样式、界面联动和特殊规则整理成文档。

建议按以下顺序推进，每完成一项即可人工验证：

1. **枚举整体主界面**
   - 标题界面：启动游戏、开始对局、退出游戏
   - 游戏战场界面：一局对战的主要操作界面
   - 结算界面：胜负产生后的结果展示与返回入口

2. **拆分战场内次要界面**
   - 持续战场区：显示城堡、建筑、士兵和战斗效果
   - 顶部状态栏：显示血量、昼夜阶段、倒计时和开发按钮
   - 手牌区：底部横向卡牌列表
   - 商店区：通过 UI 按钮打开的覆盖式界面
   - 单位/建筑信息面板：鼠标悬浮或选中对象时显示
   - 放置预览层：选择建筑卡后显示合法/非法地块
   - 结算面板：游戏结束后的高优先级覆盖层

3. **描述界面规则**
   - 简要描述每个界面的位置和样式
   - 描述界面之间的联动关系
   - 说明战场区是否支持滚动、缩放、拖动镜头等特殊规则
   - 定义覆盖式界面的输入优先级

4. **同步大纲状态**
   - 当前进度、路线图、里程碑都指向 M1.1
   - M2 保持待开发，不在本阶段实现

**完成 M1.1 的验收标准：**

- 大纲存在独立的“界面和 UI 设计”章节
- 章节中列出游戏目前有哪些整体主界面
- 章节中列出游戏战场界面有哪些次要界面
- 每个界面都有简要的位置、样式、联动和特殊规则说明
- 明确 M1.1 是文档设计阶段，暂不推进 M2 实现
