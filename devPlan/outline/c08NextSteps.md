# 八、下一步开发内容（立即行动）

**当前焦点：M3 建筑与邻接。** M2 商店、手牌与建筑卡放置已完成验收；夜晚建筑修复并入本阶段 BuildingSystem 建设时实现。

建议按以下顺序推进，每完成一项即可人工验证：

1. **BuildingSystem 抽离**
   - 从 `Castle` 抽出占格验证、放置与建筑注册逻辑
   - `CardSystem.TryPlaceCard` 与 `ShopSystem.TryPlaceOfferDirect` 改走统一放置入口
   - 为后续多建筑类型和 AI 放置预留接口

2. **多建筑类型**
   - 在现有兵营之外新增至少 2 种建筑
   - 为每种建筑补充卡牌数据、商店商品和预制体
   - 验证不同 `BuildingType` 能正确实例化并工作

3. **AdjacentSystem**
   - 实现四方向邻接检测
   - 定义首批邻接加成规则（如相邻兵营产兵加速）
   - 放置、移除或相邻建筑变化时刷新加成

4. **建筑 UI 补强**
   - 统一合法/非法地块与放置预览表现
   - 悬浮或选中建筑时显示名称、生命值等基础信息
   - 结算返回标题后确认不残留手牌、商店或放置状态

5. **夜晚建筑修复（M2 遗留）**
   - 在 BuildingSystem 就绪后，为受损建筑增加夜晚修复入口
   - 修复消耗金币，并与商店 UI 联动

**M3 启动前基线：** 标题页 → 昼夜循环 → 商店购卡/拖拽直放 → 手牌点击或拖拽放置兵营 → 结算返回标题。
