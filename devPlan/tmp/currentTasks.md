# 当前任务：制作士兵卡牌

## 目标

实现可复用的 RTS 单位卡牌 UI 组件，以卡牌代替传统士兵精灵在战场上活动，并完整接入生命值、阵营、选中、休眠、受击和死亡表现。

参考设计文档：`devPlan/design/unitCardDesign.md`

## 范围

本次包含：

- 正方形卡牌、顶部生命值条和名称、插画区、双层边框与分隔线
- 蓝色我方阵营线、红色敌方阵营线
- 选中时最外层高亮边框线及单选交互
- 状态图标与 Buff 图标的显示容器和更新接口
- 卡牌与现有士兵生成、受伤、昼夜休眠和死亡流程的绑定
- 移除旧士兵精灵、调试碰撞圈和矩形生命条

本次不包含：

- 新增 Buff、眩晕、中毒等战斗规则
- 为所有兵种制作最终插画和图标
- 改造士兵信息浮窗
- 多选、框选和编队系统
- 修改寻路、索敌和伤害规则

## 实施步骤

### 1. 建立单位卡牌预制体

- [ ] 新建单位卡牌预制体和视图脚本，放入 Godot 适配层
- [ ] 卡牌保持正方形，顶部信息区约占 18%，插画区约占 82%
- [ ] 生命值条横向铺满顶部，单位名称以白字黑描边居中覆盖在生命值条上
- [ ] 插画区包含插画、右上状态图标容器和左下 Buff 图标容器
- [ ] 实现深色粗外框、浅色细内框和两区之间唯一一条分隔线
- [ ] 阵营线位于外框内侧：我方蓝色、敌方红色，始终可见
- [ ] 选中高亮线位于卡牌最外层，未选中时隐藏且不影响布局和碰撞范围

### 2. 定义卡牌显示接口

- [ ] 支持设置单位名称、插画、卡牌尺寸和阵营
- [ ] 支持按当前生命值与最大生命值更新生命条长度
- [ ] 生命值高于 60% 使用绿色，31% 至 60% 使用黄色，30% 及以下使用红色
- [ ] 支持切换选中状态
- [ ] 支持显示一个状态图标，无状态时隐藏
- [ ] 支持水平显示最多 4 个 Buff；超过 4 个时最后一项显示 `+N`
- [ ] 没有 Buff 时隐藏整个 Buff 区

### 3. 接入单位配置与生成流程

- [ ] 使用现有单位类型、显示尺寸和阵营数据初始化卡牌
- [ ] 为当前已有单位提供显示名称映射，未制作独立插画的单位使用统一占位插画
- [ ] 将士兵预制体的旧 `Sprite2D` 表现替换为单位卡牌预制体
- [ ] 保持现有碰撞体、导航、阵营关系和生成位置不变
- [ ] 我方与敌方生成后立即显示正确的阵营线

### 4. 接入生命与状态表现

- [ ] 士兵受伤后立即刷新生命条长度和颜色
- [ ] 受击时短暂显示受击状态图标，结束后恢复当前持续状态
- [ ] 夜晚普通单位显示睡眠状态图标并保持现有变暗效果
- [ ] 夜战单位在夜晚不显示睡眠状态
- [ ] 死亡时卡牌继续使用现有缩放淡出流程
- [ ] 当前没有 Buff 数据时保持 Buff 区隐藏

### 5. 接入选中交互

- [ ] 点击士兵卡牌后选中该单位并显示最外层高亮线
- [ ] 点击另一士兵时将选中状态转移到新目标
- [ ] 点击战场空白处或选中单位死亡时取消选中
- [ ] 我方和敌方单位使用相同的选中高亮色，阵营线仍保持可见
- [ ] 选中仅改变视觉状态，不影响士兵移动、攻击和碰撞

### 6. 清理旧表现

- [ ] 移除 `SoldierLogic` 绘制的半透明碰撞圈和矩形生命条
- [ ] 移除旧士兵精灵节点
- [ ] 卡牌状态图标接管睡眠提示后，移除重复的 `zZZ` 表现
- [ ] 保持士兵信息浮窗现有行为可用

### 7. 自动验证

- [ ] `dotnet build CasualCastle.sln` 构建成功且无警告
- [ ] `dotnet test tests/CasualCastle.Domain.Tests/CasualCastle.Domain.Tests.csproj` 全部通过
- [ ] Godot 资源导入无错误，新增脚本、场景及其 UID 文件齐全

### 8. 游戏内验收

- [ ] 兵营生成的士兵以卡牌显示，不再显示裸精灵、调试圈和旧生命条
- [ ] 双方卡牌结构一致，我方蓝线、敌方红线，可在缩放视角下快速区分
- [ ] 点击单位、切换单位、点击空白和单位死亡时，选中外框状态正确
- [ ] 士兵受伤后生命条长度与颜色正确变化
- [ ] 昼夜切换时普通单位和夜战单位的状态表现符合规则
- [ ] 士兵移动、索敌、攻击、受伤和死亡流程不受卡牌替换影响
- [ ] 多个单位同屏时名称、生命值、阵营和选中状态仍清晰可读
