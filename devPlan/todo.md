## 2. 真模块化与模块内防腐

在分层基础上做**真模块化**重构：

- 各业务模块（如 `building`、`battle`、`shop`、`fusion`）**自行定义对外接口**（暴露的方法/契约），不对外泄漏内部类型。
- 模块内部通过**本模块的防腐层**调用其他模块暴露的方法，以满足本模块接口的实现需求。
- 禁止跨模块直接访问 Godot 节点、静态 `Instance` 或对方内部类；跨模块协作仅经**已暴露的接口**。

目标：模块可独立理解与替换；依赖方向清晰（接口向内，实现向外）。

## 3. 显示与缩放二次重构

在整数**游戏坐标**（每格 100 单位）已落地的前提下，进一步拆分显示职责：

1. **核心域**：按游戏坐标声明业务所需的**逻辑尺寸**（占地、碰撞、产兵点、图标占位等），不出现像素与 Godot 类型。
2. **防腐层（引擎适配）**：根据核心域给出的逻辑尺寸，用 Godot 将**源美术资源**缩放/裁切到满足该尺寸后再呈现（设计分辨率与 `ContentScale` 属于本层）。
3. **游戏内业务缩放**：在核心域之上再提供一套**与引擎无关**的缩放能力（如相邻加成、融合 tier、Buff），由规则驱动，经防腐层作用到最终显示。

目标：换图、换分辨率、换 UI 缩放策略时，不改核心域规则；业务缩放与引擎缩放职责分离。

## 4. 开发者模式

在**设置界面**增加「开发者模式」开关（持久化至 `user://` 配置，与显示设置同类）：

| 状态 | 行为 |
| --- | --- |
| **关闭（默认）** | 与正式玩家一致；隐藏开发向 UI；禁用开发向快捷键 |
| **开启** | 显示平时隐藏的 UI（调试信息、阶段跳过入口、额外日志等，清单实施时列项） |
| **开启** | 启用平时隐藏的快捷键（如 P 键作弊产兵、`DevInputLogger` 相关能力等） |

实施时注意：`GameManager` 作弊键、`DevInputLogger`、标题/战场隐藏按钮等统一经「是否开发者模式」门控，避免散落 `#if DEBUG`。

目标：正式体验干净；调试与验收不必改代码或记分支。
