# 开发大纲 — CasualCastle（中文）

## 一、概述
玩家在商店购买建筑卡牌，卡牌进入手牌。玩家可在自己的城堡中打出建筑。出兵建筑会周期性产生部队，部队在屏幕中央的战场与敌方部队交战，摧毁对方城堡获得胜利。

## 二、MVP（最小可行产品）
- 手牌系统：购买、持有、打出卡牌
- 建筑放置与生产：建筑放置后周期性产兵
- 单位战斗：简单自动寻路/直线移动并互相攻击
- 双方城堡：血量系统与胜负判定
- 简易商店界面与货币系统

## 三、核心系统模块
1. CardSystem（手牌与卡牌逻辑）
   - 购买/售卖/抽牌/手牌限制
2. BuildingSystem（建筑管理）
   - 放置、生产计时器、生成单位
3. UnitSystem（单位AI）
   - 移动、目标选择、攻击、死亡处理
4. BattleSystem（战场协调）
   - 单位生成、碰撞/伤害结算、战场状态
5. Economy（货币与商店）
6. UI（手牌、商店、城堡血条、战斗信息）

## 四、场景与资源组织
- scenes/main/ 主场景（战斗场景、UI层）
- prefabs/ 建筑与单位预制体
- assets/art/cards/ 卡牌美术
- scripts/systems/ 对应各系统的 C# 实现

## 五、里程碑（建议分阶段）
1. 项目初始化与骨架（场景、输入、基础UI）
2. 手牌与商店（购买、展示、资金）
3. 建筑放置与产兵（生产计时器、生成单位）
4. 单位战斗（攻击判定、血量、死亡）
5. 胜负判定与城堡摧毁
6. 优化与美术/音效补充

## 六、任务示例（可转为待办）
- 创建 CardSystem.cs、BuildingSystem.cs、UnitSystem.cs、BattleSystem.cs
- 设计卡牌数据资源（ScriptableObject/Resource）
- 实现商店 UI 与购买流程
- 实现建筑生产定时器与单位生成逻辑
- 实现简单单位 AI（前进→攻击最近目标）

## 七、测试与验证
- 单元测试：关键算法（生产计时、伤害计算）
- 集成测试：完整一局流程从购买到胜利
- 手动测试：多种建筑/单位交互场景

## 八、扩展方向（未来迭代）
- 建筑与单位升级、技能
- 多张战场地图与关卡设计
- 联网对战（多人同步）
- 更复杂的 AI 与路径寻路
